APEX is a digital marketing company expanding to Los Angeles, Tokyo, and Buenos Aires. This project designs the full distributed database infrastructure: a MariaDB Galera multi-master cluster with synchronous replication across all three regions.

If any single node goes down, the other two carry the load automatically. Currency, billing, and employee data stay consistent across regions at all times.

// cluster setup
# Start cluster (order matters) # Node 1 — must start first sudo galera_new_cluster # Node 2 + Node 3 sudo systemctl start mariadb # Verify cluster health sudo mysql -u root -p \ -e "SHOW STATUS LIKE 'wsrep%'"

// sample SQL (ride bookings)
-- Guest + ride bookings with JOIN SELECT g.first_name, g.last_name, r.ride_name, rb.booking_time FROM RideBookings rb JOIN Guests g ON rb.guest_id = g.guest_id JOIN Rides r ON rb.ride_id = r.ride_id; -- Average ride ratings SELECT r.ride_name, AVG(f.rating) AS avg_rating, COUNT(f.feedback_id) AS num_reviews FROM Feedback f JOIN Rides r ON f.ride_id = r.ride_id GROUP BY r.ride_name;

Two-phase enterprise network simulation in Cisco Packet Tracer. Phase 1 built the core multi-router topology with static routing. Phase 2 added full VLAN segmentation, OSPF dynamic routing, VPN tunnels for branch communication, and ASA firewall ACLs that enforce deny-by-default access policy.

// network topology
Internet │ [ASA Firewall] ← ACLs + DMZ │ [Core Router] ── OSPF ── [Branch Router 1] │ │ [Layer 3 Switch] [Branch Router 2] │ ├── VLAN 10: HR ├── VLAN 20: Finance ├── VLAN 30: Engineering └── VLAN 40: Management VPN Tunnel: HQ ◄────────────────► Branch
// security outcomes
  • 🛡VLANs isolate departments — no default cross-VLAN access
  • 🔒VPN encrypts all site-to-site traffic between branches
  • 🚫ASA ACLs deny all traffic by default — whitelist only
  • 🔄OSPF reconverges automatically on topology changes
  • 👁DMZ separates public-facing servers from internal network
